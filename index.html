1. Programming Basics

1. What is Programming?

Programming ek bhasha (language) hai jiske through hum computers ko instructions dete hain. Computer apne aap kuch nahi sochta, use step-by-step logic dena padta hai. Is logic ko code mein likhne ki process ko programming kehte hain.

2. Why do we need it?

Duniya ke saare tasks—chahe woh simple calculation ho ya advanced AI Agent ka decision-making—sab rules par aadharit hain. Manual kaam ko automate karne aur complex data ko process karne ke liye humein programming ki zaroorat hoti hai.

3. Compiler vs Interpreter (Internal Working)

Jab aap code likhte ho (High-Level Language jaise Python ya C++), computer use direct nahi samajhta. Us code ko 0s aur 1s (Machine Code) mein convert karna padta hai.

4. Variables, Constants & Comments

Variables: Variable ek container hota hai jo data store karta hai memory mein. Jaise agar aapko apna naam store karna hai, toh aap ek variable banaoge.
Constants: Wo variables jinki value program run hone ke dauran change nahi hoti (e.g., PI = 3.14).
Comments: Code ke andar wo text jo sirf developers ke padhne ke liye hota hai, interpreter use ignore kar deta hai.

Beginner Intuition
Socho ki memory ek bada sa warehouse (godown) hai. Variable us warehouse ke andar rakha hua ek dabba (box) hai, jispe aapne ek label (name) laga diya hai. Dabe ke andar ka saaman (data) aap kabhi bhi badal sakte ho!

5. Input/Output & Operators

I/O: User se data lena (Input) aur result screen par dikhana (Output).
Operators: Operators wo symbols hote hain jo variables par operations perform karte hain. Types of operators:

  • Arithmetic: +, -, *, /, //, % (Maths ke liye)
  • Comparison: ==, !=, >, < (Compare karne ke liye)
  • Logical: and, or, not (Conditions combine karne ke liye)
  • Assignment: =, +=, -= (Value assign karne ke liye)

6. Code Example: Basic Expressions & String Formatting

# Variables and Types
name = "Agent-X"  # String
version = 1.0     # Float
is_active = True  # Boolean

# String Formatting (f-strings in Python)
print(f"AI Name: {name}, Version: {version}")

# Operators
a = 10
b = 3
print(f"Addition: {a + b}")      # 13
print(f"Floor Division: {a // b}") # 3 (Removes decimal)
print(f"Modulo: {a % b}")        # 1 (Remainder)
Interview Question
Q: Python mein type conversion kaise kaam karta hai? Implicit vs Explicit kya hai?
Ans: Jab Python khud datatype badal de (e.g., int + float = float), use Implicit kehte hain. Jab developer force kare (e.g., int("10")), use Explicit ya Typecasting kehte hain.
Common Mistake
Assignment operator = ko comparison operator == ke saath confuse karna. x = 5 value assign karta hai, jabki x == 5 check karta hai ki value 5 hai ya nahi.

2. Data Types

1. What are Data Types?

Aapke paas alag-alag tarah ka data hota hai—kuch text hota hai, kuch numbers. Programming language ko batana padta hai ki data kis category ka hai, taaki memory us hisaab se allocate ho sake.

2. Internal Memory Model (Mutable vs Immutable)

Python mein har cheez ek Object hoti hai. Datatypes ko do categories mein banta gaya hai:

  • Immutable (Jo badal nahi sakte): Integers, Floats, Strings, Tuples. Agar aap inki value change karte ho, toh pichli value update nahi hoti, balki memory mein ek naya object create ho jata hai.
  • Mutable (Jo badal sakte hain): Lists, Dictionaries, Sets. Inhe memory mein usi location par modify kiya ja sakta hai.

3. Deep Dive into Collections

Lists: Ek ordered collection jo allow karti hai duplicate values aur modify ho sakti hai. Isko square brackets [] mein likhte hain.
Tuple: List ki tarah hoti hai but immutable. Ek baar ban gayi toh change nahi ho sakti. Use brackets ().
Dictionary: Key-Value pairs. Hash table data structure par based hai, isliye search bahut fast hota hai. Use curly braces {}.
Set: Unordered collection of unique items. Duplicate values automatic remove ho jati hain.

4. Code Example: Working with Collections

# List (Mutable)
ai_models = ["GPT-4", "Gemini", "Claude"]
ai_models.append("Llama-3")  # Modified in place

# Dictionary (Key-Value map)
agent_config = {
    "name": "ResearchBot",
    "temperature": 0.7,
    "tools": ["web_search", "calculator"]
}
print(agent_config["tools"])

# Set (Removes duplicates)
tokens = {"hello", "world", "hello"}
print(tokens) # Output: {'hello', 'world'}
Industry Use Case
Jab hum AI model ki settings (parameters) pass karte hain API mein, toh hamesha Dictionaries (JSON objects) ka use hota hai. Agar humein large database se unique words nikalne hain (tokenization ke time), toh hum Sets ka use karte hain taaki O(1) time complexity mile.

5. Null / None Type

Jab kisi variable ke paas koi value nahi hoti, toh usme garbage value avoid karne ke liye hum use explicitly None (Python) ya null (JS) assign karte hain. Yeh AI agents mein bahut use hota hai check karne ke liye ki kya agent ko tool_output mila ya abhi None hai.

3. Control Flow

1. What is Control Flow?

Default roop se code upar se neeche (top-to-bottom) execute hota hai. Control flow statements is execution direction ko modne (branching) ya repeat karne (looping) ke kaam aate hain. if/else statements and loops programming ki decision-making brain hain.

2. Conditional Statements (if, elif, else)

Agar koi condition True hai tabhi code block chalega. Nested conditions ka matlab ek if ke andar doosra if.

3. Loops (for, while) & Loop Control

For Loop: Jab humein pata hota hai kitni baar loop chalana hai ya jab hum kisi list/iterable ko traverse karte hain.
While Loop: Jab tak koi condition True rehti hai, loop chalta rahega. Infinite loops khatarnaak ho sakte hain.
Break: Loop ko beech mein hi completely rokne ke liye.
Continue: Current iteration ko skip karke next cycle par jane ke liye.
Pass: Ek placeholder statement jo kuch nahi karti, bas syntax error rokne ke kaam aati hai.

4. Iterators & Generators (Advanced but Crucial)

Iterator: Aisa object jiske andar state maintain hoti hai, aur next() function call karne pe agli value aati hai.
Generator: Yeh ek special function hai jo return ki jagah yield use karta hai. Iska sabse bada fayda ye hai ki ye ek sath saari memory consume nahi karta, values "on-the-fly" generate karta hai. (Lazy evaluation).

Industry Use Case for Generators
Imagine aapko 100GB ki log file AI model me process karni hai. Agar normal list use ki toh RAM crash ho jayegi. Generators (yield) use karne se file line-by-line read hogi, using minimal memory. Streaming LLM outputs (jaise ChatGPT me text ek-ek word aata hai) generators ka hi kamal hai!

5. Code Example: Generators vs Standard Logic

# Normal List (Loads everything in memory)
def get_numbers(n):
    result = []
    for i in range(n):
        result.append(i)
    return result

# Generator (Yields one item at a time - Memory Efficient)
def generate_numbers(n):
    for i in range(n):
        yield i

# Using the generator
gen = generate_numbers(5)
print(next(gen)) # 0
print(next(gen)) # 1
# Pause execution. The function remembers its state!

4. Functions

1. What are Functions?

Function block of reusable code hota hai. "Don't Repeat Yourself" (DRY) principle ko follow karne ke liye hum apne logic ko functions me wrap karte hain. Inhe parameters pass kiye jate hain, aur ye ek value return karte hain.

2. Parameters, Default Arguments & Return

Parameters vs Arguments: Function definition ke time pe jo variables likhe jate hain wo Parameters hain. Call karte time jo values pass hoti hain wo Arguments hain.
Default arguments: Agar function call karte time argument na bheja jaye, toh function default value use kar leta hai (e.g., def greet(name="User"):).

3. Scope & Closures

Scope: Variable kahan tak accessible hai (Local vs Global). Local variable function ke andar zinda rehta hai aur bahar mar jata hai.
Closure: Jab ek inner function apne outer function ke variables ko yaad rakhta hai, chahe outer function ka execution khatam hi kyun na ho gaya ho. Yeh state maintain karne ka functional tarika hai.

4. Lambda Functions & Recursion

Lambda: One-line anonymous (bina naam ke) functions. Inhe wahan use karte hain jahan function ko as an argument pass karna ho (like `map()` ya `filter()`).
Recursion: Jab function apne aap ko hi call karta hai. Base condition zaroori hai warna `RecursionError` (Stack Overflow) aayega.

5. Decorators (Python Magic)

What is it? Decorator ek function hai jo dusre function ki functionality modify ya extend karta hai bina uska code change kiye. Yeh AI/Web development (FastAPI/Flask) me routing aur logging ke liye heavily use hota hai.

graph LR A[Original Function] --> B((Decorator Wrapper)) B -->|Pre-processing| C[Execute Original Function] C -->|Post-processing| D[Return Result]
Figure 4.1: How Decorators Wrap Functions

6. Code Example: Building a Retry Decorator for APIs

import time

# A decorator to automatically retry an API call if it fails
def retry_api_call(retries=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(retries):
                try:
                    # Try executing the actual function
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"Attempt {attempt+1} failed: {e}. Retrying...")
                    time.sleep(1)
            return "All attempts failed."
        return wrapper
    return decorator

# Applying the decorator
@retry_api_call(retries=2)
def call_llm_api():
    # Simulating API failure
    raise ValueError("Network Error or Rate Limit")

# It will automatically try 2 times!
call_llm_api()
Interview Question
Q: *args and **kwargs kya hote hain?
Ans: Jab humein pata nahi hota ki function mein kitne arguments aayenge, tab hum inhe use karte hain. *args arguments ko Tuple ki tarah leta hai, aur **kwargs named arguments ko Dictionary ki tarah pass karta hai.

5. Object-Oriented Programming (OOP)

1. What is OOP?

Real-world entities (jaise Car, Bank Account, ya AI Agent) ko code mein as an "Object" model karne ke tarike ko OOP (Object-Oriented Programming) kehte hain. OOP code ko organized, modular, aur reusable banata hai.

2. Classes & Objects

Class: Ek blueprint ya naksha. Jaise "Dog" ek class ho sakti hai.
Object: Class ka real instantiation. Jaise "Tommy" ya "Sheru" us Dog class ke objects (instances) hain.

3. The 4 Pillars of OOP

  • Encapsulation: Data (variables) aur methods (functions) ko ek class ke andar bind karna aur bahar ki duniya se chupana (Private variables). Data ko protect karta hai.
  • Abstraction: Sirf zaroori details show karna, complex internal implementation hide karna. Jaise car ka steering wheel (Interface) vs Engine (Internal).
  • Inheritance: Ek class (Child) dusri class (Parent) ki properties ko inherit (le) sakti hai. Code reuse badhta hai.
  • Polymorphism: (Poly = many, morph = forms). Ek hi function ka naam alag-alag class mein alag kaam kar sakta hai (Method Overriding).

4. Constructors & Magic Methods

Constructor: (Python mein __init__). Ye tab automatically call hota hai jab object create hota hai. Initialization ke kaam aata hai.
Magic Methods (Dunder Methods): Double underscore se start hone wale special methods (e.g., __str__, __len__, __add__) jo object ka behavior native Python syntax ke sath link karte hain.

5. Code Example: Building an Agent Class

class AIAgent:
    # Constructor (Initialization)
    def __init__(self, name, memory_size):
        self.name = name                 # Public attribute
        self._memory = []                # Protected attribute
        self.__api_key = "secret_key"    # Private attribute (Encapsulation)

    # Instance Method
    def add_to_memory(self, text):
        self._memory.append(text)
        print(f"[{self.name}] Saved to memory.")

    # Magic Method for string representation
    def __str__(self):
        return f"<Agent: {self.name}>"

# Inheritance
class DevAgent(AIAgent):
    def write_code(self, language):
        print(f"{self.name} is writing {language} code...")

# Usage
coder = DevAgent("CodeBot", 1024)
coder.add_to_memory("User wants a python script.")
coder.write_code("Python")
Static vs Instance Methods
Beginners confuse @staticmethod with instance methods. Agar method class ke variables (self) ko use nahi kar raha, toh use static method banana chahiye. Yeh purely helper function ki tarah kaam karta hai class ke andar.

6. Error Handling & Debugging

1. Why do we need it?

Production mein code kabhi perfect nahi hota. API down ho sakti hai, user galat data daal sakta hai. Agar code gracefully errors ko handle nahi karega, toh application crash ho jayegi. Isse bachne ke liye exception handling use hoti hai.

2. try, except, finally blocks

  • try: Is block ke andar hum risky code rakhte hain (jo crash kar sakta hai).
  • except: Agar try block fail hua, toh program crash hone ki bajaye is block me aa jata hai. Hum specific errors pakad sakte hain (e.g., ZeroDivisionError).
  • finally: Yeh block hamesha chalega, chahe error aaye ya na aaye. Iska main use resources free karna ya file close karna hota hai.

3. Custom Exceptions & Logging

Kabhi-kabhi built-in errors kaafi nahi hote. Aap apni exceptions bana sakte hain `Exception` class ko inherit karke. Sath hi, errors ko console pe `print()` karne ki jagah `logging` module ka use karna best practice hai.

4. Code Example

import logging

# Setup basic logging
logging.basicConfig(level=logging.ERROR, format="%(asctime)s - %(message)s")

# Custom Exception
class TokenLimitError(Exception):
    pass

def generate_text(prompt_length):
    try:
        if prompt_length > 4000:
            # Raising our custom error manually
            raise TokenLimitError("Prompt too long!")
        return "Success: Generation complete."
        
    except TokenLimitError as e:
        # Handled specific custom error
        logging.error(f"Model constraint hit: {e}")
        
    except Exception as e:
        # Handled any other unexpected error
        logging.error(f"Unknown error: {e}")
        
    finally:
        print("Execution of generation pipeline finished.")

7. File Handling

1. What is File Handling?

RAM volatile hoti hai (program band hote hi data khatam). Data ko permanently store karne ke liye hum use Files (Hard Disk) me save karte hain. RAG (Retrieval-Augmented Generation) systems mein PDFs, CSVs, aur text files read karna foundation step hai.

2. Reading and Writing Files

Python mein open() function use hota hai. Context managers (with statement) ka use sabse safe hota hai kyunki file automatic close ho jati hai.

3. File Formats

  • TXT: Plain text.
  • CSV (Comma Separated Values): Tabular data. Pandas library best hai iske liye.
  • JSON (JavaScript Object Notation): Hierarchy/Nested data. Web APIs aur AI agents ka primary communication format.
  • Binary (PDF, Images): Encodings in byte format (rb/wb modes).

4. File Streaming (Handling Large Files)

Agar file 10GB ki hai toh `read()` memory ko full kar dega. Isliye hum file object pe iterators lagate hain, jo line-by-line read karta hai chunking ke through.

RAG Concept Intro
AI vector databases mein dalne se pehle, PDF files ko read karke chhote text "chunks" (jaise 500 words per chunk) mein divide kiya jata hai. File handling yahan critical role play karti hai.

8. Data Structures & Algorithms (DSA)

1. Why DSA matters in AI?

AI bas machine learning model nahi hai. Data preparation, search speed, aur system optimization directly DSA par dependent hain. Fast retrieval algorithms (like Vector Search) Graphs aur Trees par base hote hain.

2. Big O Notation (Time/Space Complexity)

Algorithm kitna "time" lega aur kitni "memory" (space) lega as data grows. Ise mathematical notation mein express karte hain: $\mathcal{O}(1)$, $\mathcal{O}(\log N)$, $\mathcal{O}(N)$, $\mathcal{O}(N^2)$.

Binary Search: Sorted list mein data dhoondhne ka sabse fast tarika $\mathcal{O}(\log N)$. Yeh har step mein data ko half kar deta hai.

9. APIs & Networking

1. What is an API?

API (Application Programming Interface) do software systems ke beech communication ka zariya hai. Jab aapka code OpenAI se connect karta hai, toh wo ek API Call hota hai.

2. HTTP Protocol & REST

Web ka base HTTP (HyperText Transfer Protocol) hai. Isme ek Request jati hai aur Response aata hai.
REST APIs 4 main HTTP methods use karte hain:

  • GET: Data lane (Read) ke liye.
  • POST: Naya data create karne ke liye (e.g., Sending prompt to LLM).
  • PUT/PATCH: Existing data update karne ke liye.
  • DELETE: Data remove karne ke liye.

10. Databases

1. What is a Database?

Structured way mein large scale data store, manage aur retrieve karne ka system. AI applications mein User profiles, agent memories aur chat history DB mein store hoti hai.

2. SQL vs NoSQL

SQL (Relational): PostgreSQL, MySQL. Data strict Tables (Rows & Columns) mein store hota hai. Relationships aur ACID transactions ke liye best hain.
NoSQL (Non-Relational): MongoDB, DynamoDB. Data JSON-like Documents mein store hota hai. Flexible schema, fast horizontal scaling. Chat logs ke liye best hai kyunki messages ka format change ho sakta hai.

3. CRUD Operations

Har database pe char (4) mukhya operations hote hain: Create (Insert), Read (Select), Update, Delete.

4. Indexing & ORM

Indexing: Jaise kitaab ke end mein index page hota hai word search karne ke liye, DB mein columns par index lagane se Search speed drastically badh jati hai.
ORM (Object-Relational Mapper): SQL queries likhne ki jagah Python Classes ka use karke DB manipulate karna (e.g., SQLAlchemy). Isse code clean rehta hai.

11. Async Programming

1. Why do we need Async?

Traditional code "Synchronous" hota hai—ek line chalti hai, jab tak wo khatam na ho, agli line nahi chalti. Agar aapka code external API (LLM) ka wait kar raha hai, toh CPU idle (khali) baitha rehta hai. Ise resolve karne ke liye Concurrency use hoti hai.

2. Threads vs Processes vs Async/Await

  • Multiprocessing: Multiple CPU cores use karna. Heavy Maths/Machine Learning tasks ke liye (CPU-Bound).
  • Multithreading: Ek process mein multiple threads (halki processing paths). File/Network I/O ke liye. But Python mein GIL (Global Interpreter Lock) ki wajah se true parallel nahi hota.
  • Async/Await (Event Loop): Single-threaded cooperative multitasking. Jab ek task (jaise Network request) wait state me jata hai, Event Loop control dusre task ko de deta hai. FastAPI aur modern web/agent systems isi pe base hain!

3. Code Example: Python Async/Await

import asyncio
import time

async def fetch_data(id, delay):
    print(f"Task {id}: Request sent...")
    await asyncio.sleep(delay) # Simulating I/O wait (Not blocking CPU)
    print(f"Task {id}: Response received!")
    return f"Data {id}"

async def main():
    # Running 3 tasks concurrently
    # Even though delays sum up to 6 seconds, this will finish in 3 seconds.
    results = await asyncio.gather(
        fetch_data(1, 2),
        fetch_data(2, 3),
        fetch_data(3, 1)
    )
    print("All Done:", results)

# Run event loop
# asyncio.run(main())
